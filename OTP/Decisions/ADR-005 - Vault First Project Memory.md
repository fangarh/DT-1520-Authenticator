# ADR-005: Vault First Project Memory

## Status

Accepted

## Context

Проект будет развиваться дольше одной сессии, а контекст агента непостоянен. Без явной базы знаний решения и ограничения будут теряться, а каждая новая задача будет начинаться с повторного исследования.

## Decision

`OTP/` используется как рабочая память проекта.

Обязательные правила:

- новые сессии начинают чтение с `Start Here`, `Current State`, `Decision Index`, `Implementation Map`
- принятые архитектурные решения фиксируются в `ADR`
- изменения статуса реализации фиксируются в `Current State`
- существенные изменения кратко пишутся в `Sessions`

## Consequences

- стоимость восстановления контекста снижается
- решения становятся видимыми и проверяемыми
- документация перестает быть случайной и становится частью рабочего процесса
