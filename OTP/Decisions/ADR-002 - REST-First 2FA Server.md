# ADR-002: REST-First 2FA Server

## Status

Accepted

## Context

Нужно обеспечить подключение к существующим решениям с минимальным порогом входа и без немедленного усложнения через корпоративные протоколы.

## Decision

Первый релиз строится как отдельный `2FA Server`, подключаемый к внешним системам через `REST API`.

Этот шаг рассматривается как первая фаза на пути к будущему `Identity mode`, но не расширяет scope первого релиза.

Расширения следующих этапов:

- `OIDC`
- `SAML`
- `RADIUS`
- `LDAP/AD` как источник пользователей и групп

## Consequences

- `API`-контракт становится критически важным долгоживущим артефактом
- можно быстрее сделать `MVP`
- enterprise-протоколы можно добавлять без переписывания ядра
- переход к более широкому identity-сценарию должен идти как отдельный следующий этап, а не как скрытое расширение `MVP`
