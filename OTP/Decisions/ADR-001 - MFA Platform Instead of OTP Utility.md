# ADR-001: MFA Platform Instead of OTP Utility

## Status

Accepted

## Context

Если строить только генератор или валидатор `OTP`, решение будет слишком узким для интеграции в реальные корпоративные сценарии и быстро упрется в ограничения безопасности и UX.

## Decision

Строим не отдельную `OTP`-утилиту, а платформу `2FA/MFA`, где `OTP` является одним из факторов.

Факторы первой очереди:

- `TOTP`
- `Push approval`
- `Backup codes`

## Consequences

- архитектура должна поддерживать несколько факторов
- мобильное приложение становится частью продукта
- интеграции проектируются на уровне платформы подтверждения, а не только проверки одноразового кода
