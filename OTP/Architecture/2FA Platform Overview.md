# 2FA Platform Overview

## Что строим

Целевой продукт: отдельная платформа `2FA/MFA`, которую можно подключать к существующим системам как внешний сервис подтверждения второго фактора.

Первый релиз - `2FA Server`. Долгосрочная стратегическая цель - развить платформу до полноценного `IdP`.

## Базовая форма продукта

- отдельный `2FA Server`
- мобильное приложение для `TOTP` и `push approval`
- административный контур
- интеграционный слой для внешних систем

## Базовые факторы первой очереди

- `TOTP`
- `Push approval`
- `Backup codes`

## Факторы второй очереди

- `WebAuthn / Passkeys`
- `SMS OTP` как ограниченный fallback
- `Email OTP` только для low-risk сценариев

## Режимы поставки

- `SaaS`
- `on-prem`
- embedded integration в существующие приложения

## Основные сценарии

1. Подтверждение входа после успешного первого фактора.
2. Step-up подтверждение чувствительных операций.
3. Подтверждение входа в enterprise-среде через адаптеры и `SSO`.
4. Подключение смартфона как постоянного второго фактора.

## Принципы

- не ограничиваться только `OTP`
- держать стабильный интеграционный контракт
- проектировать облачную и коробочную поставку из одной кодовой базы
- не завязывать критический путь только на `push`
- сохранять архитектурные границы, чтобы позже перейти от `2FA Server` к `Identity mode`
- рассматривать код и vault как два разных источника истины: реализация и решения
