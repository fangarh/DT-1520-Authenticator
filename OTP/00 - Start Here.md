# Start Here

Этот vault является рабочей базой знаний проекта.

## Как использовать

Если задача новая или контекст сессии частично потерян, читать в таком порядке:

1. [[01 - Current State]]
2. [[02 - Decision Index]]
3. [[Agent/Implementation Map]]
4. профильную заметку по домену

## Правило истины

- `OTP/` хранит проектные решения, ограничения, договоренности и операционный контекст.
- Код хранит фактическую реализацию.
- При расхождении сначала зафиксировать расхождение, потом привести vault в актуальное состояние.

## Правило обновления

После принятого решения или существенной доработки нужно:

1. обновить профильную заметку
2. обновить `Current State`, если изменился статус реализации
3. создать или скорректировать `ADR`
4. оставить короткую запись в `Sessions`

## Сейчас главное

На текущем этапе репозиторий служит как проектный каркас для будущей `2FA/MFA` платформы с опорой на `TOTP`, `push approval` и `backup codes`.

Исторические исходные заметки сохранены в `OTP/2FA/`, но каноническое чтение теперь начинается с доменных индексов.
